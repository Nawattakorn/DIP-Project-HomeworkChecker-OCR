# `checker_trocr.py` — เอกสารอธิบายโค้ด

ไฟล์นี้เป็น **โมดูลตรวจการบ้าน** เวอร์ชัน TrOCR  
รับรูปภาพใบงานของนักเรียนและเฉลย แล้วเปรียบเทียบทีละข้อ คืนคะแนนและรายละเอียดผลการตรวจ

---

## ภาพรวมการทำงาน

```
รูปภาพ + เฉลย
      │
      ▼
 extract_answers()  ← TrOCR อ่านคำตอบจากรูป
      │
      ▼
  _postprocess()    ← ทำความสะอาดคำตอบที่อ่านได้
      │
      ▼
  _normalise()      ← ปรับรูปแบบให้เหมือนกันก่อนเทียบ
      │
      ▼
   เปรียบเทียบ + คำนวณคะแนน
      │
      ▼
     dict ผลลัพธ์
```

---

## ฟังก์ชัน `_normalise(text)`

### รับ
| พารามิเตอร์ | ชนิด | คำอธิบาย |
|---|---|---|
| `text` | `str` | ข้อความดิบที่ต้องการปรับรูปแบบ |

### ทำ
1. แปลง `text` เป็น string (กันกรณี input ไม่ใช่ str)
2. ตัด whitespace หัว-ท้ายออก (`.strip()`)
3. ลบ whitespace ที่อยู่ตรงกลางทั้งหมดออกด้วย regex `\s+`
4. แปลงตัวอักษรทั้งหมดเป็นตัวพิมพ์ใหญ่ (`.upper()`)

### ส่ง
`str` — ข้อความที่ไม่มีช่องว่าง เป็นตัวพิมพ์ใหญ่ทั้งหมด

### เพราะ
การเปรียบเทียบคำตอบกับเฉลยต้องไม่สะดุดเรื่องช่องว่างหรือตัวพิมพ์เล็ก/ใหญ่  
เช่น `"  hello "` กับ `"HELLO"` ต้องถือว่าเหมือนกัน

## ฟังก์ชัน `_postprocess(raw, expected)`

### รับ
| พารามิเตอร์ | ชนิด | คำอธิบาย |
|---|---|---|
| `raw` | `str` | ข้อความดิบที่ TrOCR อ่านได้จากรูป |
| `expected` | `str` | เฉลยของข้อนั้น ใช้เพื่อตัดสินว่าต้องจัดการ `raw` แบบไหน |

### ทำ
1. Normalize ทั้ง `raw` และ `expected` ด้วย `_normalise()`
2. **ตรวจสอบชนิดของเฉลย:**
   - **ถ้าเฉลยเป็นตัวเลขล้วน** (`isdigit()`)  
     → ดึงเฉพาะตัวเลขออกจาก `raw_norm` ด้วย regex `\d+`  
     → ใช้ตัวเลขตัวสุดท้ายที่พบ (หากไม่พบเลย ใช้ `raw_norm` เดิม)
   - **ถ้าเฉลยเป็นข้อความ/ภาษาอังกฤษ**  
     → ลบทุกอักขระที่ไม่ใช่ `A-Z` หรือ `0-9` ออก

### ส่ง
`str` — คำตอบที่ทำความสะอาดแล้ว พร้อมนำไปเทียบกับเฉลย

### เพราะ
TrOCR อาจอ่านได้ปนสัญลักษณ์พิเศษ เช่น `"42."` หรือ `"AN-SWER"`  
การ postprocess ช่วยให้การเปรียบเทียบแม่นยำขึ้น

## ฟังก์ชัน `grade(image_path, key_path, provided_key)`

ฟังก์ชันหลักของโมดูล รวมทุกขั้นตอนตั้งแต่รับรูปจนได้คะแนน

### รับ
| พารามิเตอร์ | ชนิด | คำอธิบาย |
|---|---|---|
| `image_path` | `str` | path รูปภาพใบงานที่ต้องการตรวจ |
| `key_path` | `str` \| `None` | (สำรอง, ยังไม่ได้ใช้งาน) path ไฟล์เฉลย |
| `provided_key` | `list` \| `None` | รายการเฉลย เช่น `["42", "HELLO", "3"]` |

### ทำ
**ขั้นที่ 1 — เตรียมเฉลย**
- แปลง `provided_key` → list ของ dict `{"id": ข้อที่, "answer": เฉลย}`
- ถ้าไม่มี `provided_key` → ตั้ง `questions = []`

**ขั้นที่ 2 — อ่านคำตอบจากรูปด้วย TrOCR**
- เรียก `extract_answers(image_path)` จาก `ocr_trocr.py`
- ได้ `raw_lines` เป็น list of str แต่ละบรรทัดคือคำตอบแต่ละข้อ
- หากเกิด exception → คืน error dict ทันที

**ขั้นที่ 3 — ปรับขนาด list ให้ตรงกับจำนวนข้อ**
- Pad ด้วย `""` หากอ่านได้น้อยกว่าจำนวนเฉลย
- Trim หากอ่านได้มากกว่าจำนวนเฉลย

**ขั้นที่ 4 — ตรวจทีละข้อ**
```
สำหรับแต่ละข้อ i:
  expected      = เฉลยข้อ i
  expected_norm = _normalise(expected)
  got           = _postprocess(raw_lines[i], expected)

  ถูก ถ้า: expected_norm == got
       หรือ: expected_norm เป็น substring ของ got  ← "ใจกว้าง"
```
- เก็บผลลัพธ์รายข้อเป็น dict `{id, expected, got, correct}`

**ขั้นที่ 5 — คำนวณคะแนน**
```
score = (จำนวนข้อถูก / จำนวนทั้งหมด) × 100
```

### ส่ง

**กรณีสำเร็จ:**
```json
{
  "success": true,
  "score": 75.0,
  "correct": 3,
  "total": 4,
  "results": [
    {"id": 1, "expected": "42", "got": "42", "correct": true},
    ...
  ],
  "extracted_raw": ["42", "hello", ...]
}
```

**กรณี error (OCR ล้มเหลว):**
```json
{
  "success": false,
  "error": "...",
  "score": 0,
  "results": []
}
```

### เพราะ
เป็น entry point หลักที่ Flask app เรียกใช้ รวม logic ทั้งหมดไว้ในที่เดียว  
ทำให้ controller ไม่ต้องรู้รายละเอียดภายใน
